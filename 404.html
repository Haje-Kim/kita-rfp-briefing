Background

추진 배경과 전략적 의의

03 / 13

분산된 무역 데이터·서비스를 통합하고, AI Agent 기반 지능형 수출지원 플랫폼으로 전환한다.

왜 지금인가

글로벌 무역 환경 변화(ESG·관세·디지털 무역)와 회원사 요구 수준 상승. 기존 사일로 구조로는 대응 한계.

무엇을 바꾸나

산재된 데이터를 통합데이터플랫폼(Trade Digital Brain)으로 일원화, 대화형 Multi-Agent 환경으로 전 서비스 재구성.

어디까지 가나

수출 준비 → 디지털 무역 연계 → 사후관리까지 E2E 원스톱 지원. 무역 생태계 허브 역할 정착.

사업 목표(3축)

  • Trade Agent — 무역 특화 지능형 대화형 AI 서비스로 업무 전주기 지원
  • 원스톱 E2E 통합 서비스 — tradeKorea·수출실적증명·무역아카데미·ESG·지재권 AI 도입 및 고도화
  • 클라우드 인프라 — 랜딩존·공통 운영환경·보안체계 기반 고가용성·확장성 확보

Scope of Work — Phase 1

1단계 추진범위 — 8개 핵심 과제

04 / 13

본 사업의 과업 경계. 8개 과제는 상호 연결되지만 평가·산출물 단위는 과제별로 구분된다.

1Trade Digital Brain

Data Lakehouse 기반 통합데이터플랫폼. 데이터 파이프라인·표준화·메타 관리·리니지·무역통계 전환 구축.

2Trade Agent

Multi-Agent 통합 서비스. 의도·맥락 기반 오케스트레이션, 도메인 특화 Agent, RAG·벡터화.

3맞춤형 통합대시보드

HS코드·관심국가·세그먼트 기반 개인화 위젯, 추천·알림(SMS·메신저·App Push).

4KITA 멤버십 고도화

수출실적증명 신청·발급 AI 자동화(서류 추천·검증·리포트·반려문안 생성).

5글로벌 지재권 서비스

지재권 확보·관리·활용 통합 지원. 유관기관 연계·분류체계·특화 콘텐츠·전담 화면.

6tradeKorea + 전자무역

AI 바이어/셀러 매칭, 모바일 App 재구축, uTradeHub·LogisHub·PTB·해외세관 연계 설계.

7ESG 규제대응 플랫폼

EU CSRD·CBAM·VSME 대응 진단, 약식 ESG 보고서(XBRL), 맞춤 규제정보·AI 상담.

8차세대 무역아카데미

LMS·관리자 포털·모바일 학습·AI 튜터·KNEX/KAMIS 연계.

※ 각 과제 산출물·요구사항(SER·FUR·DAR·TER 등)은 RFP Ⅲ장 참조. 8개 과제 모두 본 1단계 사업 범위.

Target Architecture Principles

TO-BE 아키텍처 원칙

07 / 13

RFP는 구체적 블록도가 아닌 원칙·요구사항 수준으로 TO-BE를 정의. 아래 6개 원칙이 설계·평가의 축.

01
Data Lakehouse 일원화
산재 데이터를 통합 데이터 플랫폼으로 모으고 AI Data Hub 구성. 표준화·메타·리니지 일관 적용.
02
Multi-Agent Orchestration
의도·맥락 파싱 → 작업 흐름 조정 → 도메인 Agent 호출. Master/Orchestration/도메인 Agent 계층화.
03
Cloud Native + 랜딩존
클라우드 랜딩존·공통 운영환경·보안체계. 확장성·민첩성·고가용성 확보.
04
SSO·통합 인증
KITA.net SSO 중심 인증 통합. 회원사·비회원사·관리자 경로 분리. MFA·토큰 갱신 정책 강화.
05
개인화 · 추천 · 알림
관심분야·행동로그 기반 개인화. SMS/메신저/App Push 다채널 알림 체계.
06
표준 · 보안 · 시큐어코딩
전자정부 프레임워크 호환, LLM 데이터 접근 보안, Agent 보안 취약점 점검, 시큐어코딩 의무.

  • PL은 주사업자 또는 공동수급체 소속 직원

  • 하도급 시 발주기관 사전 승인 필수

  • 재하도급 원칙적 불허

  • 타 과제 중복 투입 인력은 기술평가 대상 제외
  • "~할 수 있다" 등 모호 표현은 불가능으로 간주

성능 및 품질성능·품질·인터페이스(시스템·사용자) 요구15
프로젝트 관리관리방법론·일정 계획·개발 장비15
프로젝트 지원품질보증·시험운영·교육·유지관리·하자보수·기밀보안·비상대책20

가격 평가

  • 가격 점수 = (최저가격 / 평가대상자 제안가격) × 배점

제약·유의사항 (3가지)

12 / 13

자주 이슈되는 조건 3가지. 제안서 작성·내부 의사결정 전에 반드시 확인.

① 보안 — 인프라 구성도 비공개

현 인프라 구성도는 보안 지침으로 RFP에 미포함. 제안서 작성에 필요 시 보안서약서 작성 후 협회 방문으로만 열람 가능. 일정 여유 확보 필요.

② 상주 원칙 + 인력 변경 제약

투입인력은 상주가 원칙. 비상주 사유 불명확 시 공수 제외. 제안 인력 임의 변경 불가, 교체 시 동급 이상·5일 인계인수 의무·교체 인건비 사업자 부담.

③ 표현·산출물 엄격성

"~할 수 있다·예정이다·고려한다" 등 모호 표현은 불가능으로 간주. 제안 설명회 시 PM 직접 발표 필수. 제출 산출물·솔루션은 상거래·법률 하자 없을 것.

기타 핵심 의무

  • 지적재산권 — 본 사업 산출물 소유권은 협회로 귀속
  • 하자보증 — 검사완료일로부터 1년
  • 보안각서 — 투입 전원 제출, 정기 보안교육
